## 2-Tap FFE 與一階 RC 通道之完美等化

### 數學推導
1. **一階 RC 通道模型**：
   假設通道為簡單的一階 RC 低通濾波器，其步階響應 (Step Response) 為 $s(t) = 1 - e^{-t/\tau}$，其中 $\tau = RC$ 為時間常數。
2. **單一脈衝響應 (Pulse Response)**：
   傳送一個寬度為 $T_b$ (1 UI) 的脈衝，可視為兩個步階函數相減：
   $p(t) = s(t) - s(t-T_b)$
3. **取樣點的 ISI 分析**：
   - 主訊號 Main-cursor (在 $t=T_b$ 取樣)：$p(T_b) = 1 - e^{-T_b/\tau}$
   - 第一個 Post-cursor (在 $t=2T_b$ 取樣)：
     $p(2T_b) = s(2T_b) - s(T_b) = (1 - e^{-2T_b/\tau}) - (1 - e^{-T_b/\tau}) = e^{-T_b/\tau}(1 - e^{-T_b/\tau})$
4. **2-Tap FFE 的 Zero-Forcing 條件**：
   加入 2-Tap FFE，權重為 $w_0 = 1$、$w_1 = \alpha$。等化後的脈衝響應 $h_{FFE}(t) = p(t) + \alpha \cdot p(t-T_b)$。
   為了強迫消除第一個 Post-cursor 的 ISI，我們令 $h_{FFE}(2T_b) = 0$：
   $p(2T_b) + \alpha \cdot p(T_b) = 0$
   代入上式：$e^{-T_b/\tau}(1 - e^{-T_b/\tau}) + \alpha (1 - e^{-T_b/\tau}) = 0$
   消去公因式 $(1 - e^{-T_b/\tau})$ 後得到：$\alpha = -e^{-T_b/\tau}$
5. **連續相同位元（平坦區間）的物理驗證**：
   觀察圖中 $x(t)$ 在 $2T_b \sim 4T_b$ 傳送連續兩個 '1'。
   - 在 $t=3T_b$ 時，電容電壓 $y(3T_b)$ 已充至 $1 - e^{-T_b/\tau}$。
   - 在 $3T_b \sim 4T_b$ 區間，$x(t)=1$ 且 $x(t-T_b)=1$，此時 FFE 輸出 $x_2(t) = 1 + \alpha = 1 - e^{-T_b/\tau}$。
   - 驚人的是，RC 電路的「目標輸入電壓 $x_2(t)$」剛好等於「當前電容電壓 $y(3T_b)$」，所以電容不充也不放，$y(t)$ 波形在綠色圖中呈現完美的水平線！這證明了穩態下的 ISI 也被完美消除。

### 單位解析
**公式單位消去：**
- **時間常數 $\tau = R \times C$**：$[ \Omega ] \times [ F ] = [ V/A ] \times [ A \cdot s / V ] = [ s ]$ (秒)
- **FFE 權重 $\alpha = -e^{-T_b/\tau}$**：指數的次方項 $-T_b/\tau$ 單位為 $[ s ] / [ s ] = [ 1 ]$ (無因次)。因此 $\alpha$ 是無因次 (Dimensionless) 的電壓比例常數。
- **等化後輸出 $x_2(t) = x(t) + \alpha \cdot x(t-T_b)$**：$[V] + [1] \times [V] = [V]$ (電壓)。

**圖表單位推斷：**

### 白話物理意義
在資料發生轉態時，透過 FFE 故意給訊號一記「過猛的電壓變化 ($\alpha$)」，剛好抵銷掉 RC 通道充放電太慢的拖延症，讓訊號在取樣瞬間能分毫不差地到達目標值。

### 生活化比喻
就像開車（訊號）遇到充滿泥濘的爛路（RC 通道），車子起步和煞車都會拖泥帶水。FFE 就像是你在起步時故意「重踩油門」，要煞車時「重踩煞車」，用人為的過度操作，剛好抵銷泥濘的阻力，讓你能在準確的時間點完美停在斑馬線上。

### 面試必考點
1. **問題：為什麼筆記上特別用紅字寫「RC model 過度簡化, 不真實 (impossible)」？** 
   → **答案：** 因為真實的 PCB 通道具有傳輸線效應、趨膚效應、介電損耗、多重反射等，屬於「多階極點與零點」的複雜系統。單一 RC 模型會讓人誤以為區區 2-Tap FFE 就能產生「No Jitter, No ISI」的完美眼圖，這在實際高速 SerDes 中是絕對不可能的，實務上通常還需要 DFE 或 CTLE 輔助。
2. **問題：在 FFE 中，為什麼 tap weight $\alpha$ 通常是負的？** 
   → **答案：** 為了執行 De-emphasis (去強調)。當連續相同 bit 出現時（缺乏高頻成分），我們降低其穩態振幅至 $1+\alpha$；而在轉態時（富含高頻成分）保持最大振幅 $1$，以此等效「提升高頻、衰減低頻」，來補償通道的低通衰減特性。
3. **問題：若通道變長 (Loss 變大)，$\alpha$ 的絕對值應該如何調整？** 
   → **答案：** 需變大。通道變長代表等效 RC 時間常數 $\tau$ 變大，系統頻寬變小。由推導出的公式 $\alpha = -e^{-T_b/\tau}$ 可知，$\tau$ 變大會使 $e^{-T_b/\tau}$ 變大，因此需要更強的補償力度（更大的負值 $|\alpha|$）才能把殘留的 ISI 給消除。

**記憶口訣：**
一階 RC 太天真，兩 Tap 就能定乾坤；真實通道多極點，完美眼圖只能夢。
