## 雙模除頻器 (Multi-Modulus Dividers / Prescalers)

### 數學推導
本頁筆記探討如何利用兩個 D-Type Flip-Flop (DFF) 與基本邏輯閘，建構出可切換除數的 $\div 2 / 3$ 雙模除頻器。這是合成小數 N (Fractional-N) 頻率合成器的核心基石。

**第一種架構分析：**
定義節點與變數：
- $Q_1$ 為第一級 DFF (左) 的輸出，即筆記標示的節點 `D`。
- $Q_2$ 為第二級 DFF (右) 的輸出，其反相輸出 $\overline{Q}_2$ 即為節點 `C`。
- 輸入控制訊號為 $M$。

推導電路布林代數方程式：
1. 第一級 DFF 輸入：$D_1 = C = \overline{Q}_2$
2. OR Gate 輸出：$A = Q_1 + M$
3. AND Gate 輸出 (第二級 DFF 輸入)：$D_2 = B = A \cdot C = (Q_1 + M) \cdot \overline{Q}_2$

**情境一：當 $M = 1$ ($\div 2$ 模式)**
- 由於 $M=1$，OR gate 輸出 $A = Q_1 + 1 = 1$。
- 第二級輸入變成 $D_2 = 1 \cdot \overline{Q}_2 = \overline{Q}_2$。
- **結論**：這使得第二級 DFF 退化成一個單純的 Toggle Flip-Flop (每次 Clock 狀態反轉一次)。第一級 DFF 的輸出被 OR gate 屏蔽 (Mask) 掉了。因此，輸出 `ckout` ($Q_2$) 的頻率直接是輸入 `ckin` 的一半，實現 **$\div 2$**。

**情境二：當 $M = 0$ ($\div 3$ 模式)**
- 由於 $M=0$，OR gate 輸出 $A = Q_1 + 0 = Q_1$。
- 第二級輸入變成 $D_2 = Q_1 \cdot \overline{Q}_2$。
- 我們來推導狀態機 $(Q_1, Q_2)$ 的轉移過程：
  - 狀態方程式：$Q_1[n+1] = \overline{Q}_2[n]$， $Q_2[n+1] = Q_1[n] \cdot \overline{Q}_2[n]$
  - 初始狀態設為 $(0,0)$：
    - Cycle 1: $\overline{Q}_2=1 \Rightarrow$ 下一個狀態 $Q_1 = 1$, $Q_2 = 0 \cdot 1 = 0$。轉移至 **$(1,0)$**。
    - Cycle 2: $\overline{Q}_2=1 \Rightarrow$ 下一個狀態 $Q_1 = 1$, $Q_2 = 1 \cdot 1 = 1$。轉移至 **$(1,1)$**。
    - Cycle 3: $\overline{Q}_2=0 \Rightarrow$ 下一個狀態 $Q_1 = 0$, $Q_2 = 1 \cdot 0 = 0$。轉移至 **$(0,0)$**。
- **結論**：系統在 $(0,0) \rightarrow (1,0) \rightarrow (1,1)$ 三個狀態中無限循環。因為需要 3 個 Clock 週期才能完成一次完整狀態循環，輸出頻率為輸入的 1/3，實現 **$\div 3$**。

**Alternative $\div 2 / 3$ 架構分析 (下方電路)：**
- 方程式：$D_1 = \overline{Q}_1 + (M \cdot \overline{Q}_2)$， $D_2 = Q_1$
- 當 $M=0$：$D_1 = \overline{Q}_1$，FF1 變為 Toggle FF ($\div 2$)，FF2 僅為 Delay，整體為 **$\div 2$**。
- 當 $M=1$：$D_1 = \overline{Q}_1 + \overline{Q}_2$。狀態轉移 $(Q_1, Q_2)$ 會依序走過 $(1,0) \rightarrow (1,1) \rightarrow (0,1)$ 三個狀態循環，整體為 **$\div 3$**。

### 單位解析
**公式單位消去：**
除頻器的核心公式為：$f_{out} = \frac{f_{in}}{N}$
- $f_{in}$ [Hz] = [1/s] (輸入時脈頻率)
- $N$ [無單位] (除數，此處為 2 或 3)
- $f_{out} [Hz] = \frac{[1/s]}{[無單位]} = [1/s] = [Hz]$

**圖表單位推斷：**
📈 筆記左上方的 Timing Diagram 單位推斷：
- **X 軸**：時間 [ps] 或 [ns]。在高速 SerDes 應用中，若輸入時脈為 10 GHz，則一個週期為 100 ps，故 X 軸典型範圍為 0 ~ 1000 ps。
- **Y 軸**：電壓 [V]。通常代表內部數位邏輯的 High/Low 準位，典型範圍為 0 ~ VDD (例如 0V ~ 0.9V)。

### 白話物理意義
雙模除頻器就像腳踏車的變速齒輪，控制訊號 M 是變速撥桿；撥到一邊時踩 2 圈輪子轉 1 圈 ($\div 2$)，撥到另一邊時要踩 3 圈輪子才轉 1 圈 ($\div 3$)，藉由不斷切換這兩種檔位，我們就能創造出「平均踩 2.4 圈轉 1 圈」的小數除頻效果。

### 生活化比喻
想像你在便利商店集點換咖啡。
- $M=1$ 時你是 **VIP 會員**：每集滿 **2 點**就能換一杯 (Flip-Flop Toggle，$\div 2$)。
- $M=0$ 時你是 **普通會員**：必須集滿 **3 點**才能換一杯 (狀態機繞三圈才回到原點，$\div 3$)。
電路中的邏輯閘 (AND/OR Gate) 就是店員，根據你出示的會員卡 (M) 決定要不要扣掉你的點數並給你咖啡。

### 面試必考點
1. **問題：在高速 SerDes 的 PLL 中，為何我們需要 Multi-Modulus Divider 而不是簡單的計數器？**
   - **答案**：為了實現 Fractional-N (小數分頻) PLL。高頻的 VCO 輸出無法直接讓大位元的計數器跑起來，必須先用 Prescaler (如 $\div 2/3$ 或 $\div 4/5$) 將頻率降下來。透過動態切換除數 (例如長時間中 60% 除以 2，40% 除以 3)，就能得到平均 $2.4$ 的小數除頻比，大幅提高頻率合成的解析度。

2. **問題：請看上方第一種電路，若現在頻率操作在非常高的極限，這個架構會有什麼隱患？**
   - **答案**：**Critical Path (關鍵路徑) 延遲過長**。在第一種架構中，從 FF1 的 Q 端輸出，必須經過一個 OR Gate 再經過一個 AND Gate，才能到達 FF2 的 D 輸入端。在極高頻 (如幾十 GHz) 時，這兩個邏輯閘的 Gate Delay 可能會導致訊號來不及在下一個 Clock Edge 建立穩定，造成 Setup Time Violation。

3. **問題：基於上述缺點，筆記下方的 Alternative 架構優勢為何？**
   - **答案**：Alternative 架構重新分配了邏輯閘的位置。它將組合邏輯移到了 FF1 的輸入端 ($D_1$)。這樣設計的目的是為了優化佈局 (Layout) 或是利用特定製程庫中優化過的複合邏輯閘 (如 AOI/OAI gate)，將 FF 之間的邏輯深度降到最低，從而提升電路能操作的最高頻率 ($f_{max}$)。

**記憶口訣：**
「**M控雙模二與三，1是Toggle單純翻；0吞脈衝繞三圈，高頻尤看Gate延遲。**」
